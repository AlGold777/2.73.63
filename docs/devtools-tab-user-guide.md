# Инструкция: вкладка DevTools (Logs / Telemetry / Selectors / API)

Этот гайд для пользователя, который впервые открыл окно DevTools нашего расширения.

## 1. Как открыть
- На странице модели (ChatGPT/Claude/Grok и т.д.) нажмите иконку расширения → кнопку **DevTools**.  
- Появится модальное окно с вкладками: **Logs**, **Telemetry**, **Selectors**, **API**.

## 2. Вкладка Selectors
- **Selector Health**: показывает живую картину работы селекторов по каждой модели/элементу. Вы увидите:  
  - распределение L1–L4, долю fail/error, общее число семплов и время последнего обновления;  
  - **Last override** — когда и почему менялся селектор;  
  - **Last check** — результат последней проверки (OK/FAIL + слой);  
  - **Active UI** — текущая версия UI на активной вкладке модели (если вкладка открыта).  
  Если данных нет — подпись «нет данных» в заголовке; кнопка ↻ справа перезапрашивает статус.
- Кнопки в заголовке Selector Health: **⧉ Copy** копирует таблицу, **{ } Export** сохраняет JSON.
- **Live selector editor** (справа от Selector Health):
  - Выберите **Model** (платформу).
  - Выберите **Element** (composer / sendButton / response).
  - **Active UI version**: показывает текущую версию UI на активной вкладке модели; ↻ обновляет статус.
  - **Active selectors**: список селекторов для текущей версии UI с возможностью проверить совпадения.
  - В списке ниже увидите активные селекторы.  
  - В поле введите новый CSS‑селектор:
    - **👀 Highlight** — подсветит найденные элементы на текущей странице (ничего не сохраняет).
    - **✅ Validate** — проверит селектор на активной вкладке и покажет количество совпадений.
    - **💾 Save override** — сохранит селектор как override для выбранной модели/элемента; после сохранения новые запросы используют его.
  - Перед сохранением требуется **Причина / тикет** — это фиксируется в истории.
- **Override History**: журнал последних изменений селекторов с временем, моделью и причиной.

## 3. Вкладка Logs
- Показывает диагностические логи только для моделей, выбранных в основном окне.  
- Кнопки в шапке:
  - **⧉ Copy all** — копирует видимые записи в буфер обмена.
  - **🗑️ Clear** — очищает текущие записи на панели (локально).

Что важно для вкладок/отправки:
- События `DISPATCH` — это фон: активация вкладки модели, ожидание готовности и отправка `GET_ANSWER`.
- Событие `PROMPT_SUBMITTED` — это контент: “Send реально нажался”, после него фон может безопасно перейти к следующей вкладке.

## 4. Вкладка Telemetry
- Список платформ содержит только модели, выбранные в главном окне.  
- **Copy** — копирует видимые события телеметрии в буфер обмена.  
- **{ } Export** — сохраняет JSON, где события сгруппированы тегами моделей (например, `<GPT>`).  

## 5. Вкладка API
- Поля для ключей: OpenAI, Claude, Gemini, Grok.  
- Введите ключи при необходимости, они сохраняются локально в браузере.

## 6. Частые вопросы
- **Не вижу данных в Selector Health**: дождитесь загрузки или нажмите ↻. Если платформа закрыта, данных не будет.  
- **Подсветка селектора не работает**: убедитесь, что выбрана корректная модель и элемент; попробуйте другой CSS‑селектор.  
- **Логи пустые**: расширение пока не выполняло действий или фильтр скрывает записи.
- **После перезагрузки расширения интерфейс “помнит” старые размеры**: при повторной инициализации геометрия (ширина сайдбаров/высота панелей) автоматически сбрасывается к дефолту.

## 7. Безопасность и хранение
- Все настройки/ключи сохраняются локально в браузере (chrome.storage/local или localStorage).  
- Ничего не отправляется наружу без вашего запроса.
