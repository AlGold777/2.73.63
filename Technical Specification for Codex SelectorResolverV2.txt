Technical Specification for Codex
SelectorResolverV2 + ResponseLifecycleDetector for LLM Browser Extension
0. Task name
Implement deterministic Skyvern-style DOM-spatial self-healing selector layer and response lifecycle detection for multi-LLM browser extension automation.

1. Project context
We are developing a browser extension that sends one user prompt to multiple LLM web interfaces at the same time and collects their answers.
Supported platforms:
ChatGPT / GPT
Claude
Gemini
Grok
Le Chat
Qwen
DeepSeek
Perplexity
Current flow:
User enters prompt
→ selects LLMs
→ extension opens or reuses LLM tabs
→ background dispatches prompt to each tab
→ content scripts inject and send prompt
→ extension waits while models generate responses
→ user clicks Get Answers or auto-collection runs
→ extension collects answers
→ answers are displayed in result columns
The existing system already has:
background dispatch coordinator
job orchestrator / staged rounds
tab manager
message router
content scripts per model
selector configs per model
diagnostics / telemetry
human-presence simulation
answer collection logic
This task must improve resilience when LLM websites change their DOM/UI.

2. Core architecture decision
Do not integrate full Skyvern runtime.
Do not add Playwright, Python, local server, external VLM, or API-based vision.
Implement a free deterministic DOM-spatial fallback layer inspired by Skyvern.
The final architecture must have two separate modules:
SelectorResolverV2
  answers: WHERE is the element?

ResponseLifecycleDetector
  answers: WHEN did the answer start and finish?
The existing background orchestrator and dispatch coordinator remain the master process controllers.

3. Non-goals
Codex must not implement:
- full Skyvern integration
- Playwright automation
- Puppeteer automation
- Python companion server
- external API calls
- GPT/Claude/Gemini vision calls
- screenshot analysis
- CAPTCHA bypass
- anti-bot bypass
- stealth automation
- coordinate-based primary automation
- full rewrite of model content scripts

4. Hard constraints
Codex must preserve:
- existing dispatch flow
- existing GET_ANSWER / GET_ANSWER_NO_FOCUS semantics
- existing PROMPT_SUBMITTED semantics
- existing LLM_RESPONSE / FINAL_LLM_RESPONSE semantics

Codex must not:

PART A — SelectorResolverV2

6. SelectorResolverV2 purpose
SelectorResolverV2 must provide robust fallback resolution for:
- composer / prompt input
- send button
- latest assistant answer container
It must implement layered resolution:
Tier 0 — exact selectors from existing config
Tier 1 — validated cached selector
Tier 2 — semantic DOM candidates
Tier 3 — spatial heuristics
Tier 4 — disabled visual resolver placeholder only
Tier 4 must not make screenshots or external requests in this task.

function extractSafeVisibleText(el, maxChars = 10000) {
  if (!el) return '';

  let text = '';

  try {
    text = el.innerText || el.textContent || '';
  } catch (_) {
    text = '';
  }

  return String(text)
    .replace(/\s+/g, ' ')
    .trim()
    .slice(0, maxChars);
}

16. Deep DOM / Shadow DOM scanning
Shadow DOM scanning is mandatory.
Implement conservative open-shadow scanner:
function queryAllDeep(root, selectors, maxShadowDepth = 2, hasBudget = null) {
  const selectorText = Array.isArray(selectors) ? selectors.join(',') : selectors;
  const results = [];
  const visited = new Set();

  const MAX_SHADOW_HOST_SCAN_NODES = 500;
  const MAX_TOTAL_RESULTS = 300;

  function scan(node, depth) {
    if (!node || visited.has(node)) return;
    if (typeof hasBudget === 'function' && !hasBudget()) return;
    if (results.length >= MAX_TOTAL_RESULTS) return;

    visited.add(node);

    try {
      if (typeof node.querySelectorAll === 'function') {
        const found = Array.from(node.querySelectorAll(selectorText));
        for (const el of found) {
          if (results.length >= MAX_TOTAL_RESULTS) break;
          results.push(el);
        }
      }
    } catch (_) {}

    if (depth >= maxShadowDepth) return;
    if (typeof hasBudget === 'function' && !hasBudget()) return;

    let all = [];
    try {
      if (typeof node.querySelectorAll === 'function') {
        all = Array.from(node.querySelectorAll('*'));
      }
    } catch (_) {
      all = [];
    }

    if (all.length > MAX_SHADOW_HOST_SCAN_NODES) {
      all = all.slice(0, MAX_SHADOW_HOST_SCAN_NODES);
    }

    for (const el of all) {
      if (typeof hasBudget === 'function' && !hasBudget()) break;

      try {
        if (el.shadowRoot) {
          scan(el.shadowRoot, depth + 1);
        }
      } catch (_) {}
    }
  }

  scan(root || document, 0);

  return [...new Set(results)].filter(Boolean);
}
Rules:
- scan normal document first
- scan open shadow roots up to depth 2
- ignore closed shadow roots
- ignore inaccessible iframes
- wrap all shadow/iframe access in try/catch
- never throw from scanner
- coordinates from getBoundingClientRect() are treated as viewport coordinates
- all resolver calls must pass hasBudget into queryAllDeep()

17. Resolver timeout budget
timeoutMs is a hard wall-clock budget for one resolver call.
It applies to:
- exact selector attempts
- cache validation
- deep candidate scan
- candidate snapshot building
- scoring
- telemetry preparation
It is not a long polling mechanism.
Implement:
const startedAt = Date.now();

function hasBudget() {
  return Date.now() - startedAt < timeoutMs;
}
Each expensive stage must check:

function buildCandidateSnapshot(el) {
  let rect = null;
  let style = null;

  try {
    const r = el.getBoundingClientRect();
    rect = {
      top: r.top,
      left: r.left,
      right: r.right,
      bottom: r.bottom,
      width: r.width,
      height: r.height
    };
  } catch (_) {}

  try {
    const cs = getComputedStyle(el);
    style = {
      display: cs.display,
      visibility: cs.visibility,
      opacity: cs.opacity,
      pointerEvents: cs.pointerEvents,
      overflowY: cs.overflowY,
      position: cs.position,
      zIndex: cs.zIndex

  } catch (_) {}

  return {
    el,
    rect,
    style,
    tagName: el.tagName,
    role: el.getAttribute?.('role') || '',
    type: el.getAttribute?.('type') || '',
    ariaLabel: el.getAttribute?.('aria-label') || '',

    placeholder: el.getAttribute?.('placeholder') || '',
    dataTestId: el.getAttribute?.('data-testid') || '',
    text: getAccessibleText(el)

26. Stable selector strategy
Preference order:
1. stable id if unique
2. data-testid / data-test / data-cy if unique
3. aria-label + role/tag if unique
4. placeholder + tag if unique
5. role + nearby parent if unique
6. nth-of-type path as last resort, max depth 5
Do not generate selectors longer than 300 chars.
Generate max 12 selector candidates per element.
Use bounded uniqueness helper.
function isUniqueSelector(selector, root = document, limit = 1) {
  if (!selector || selector.length > 300) return false;

  try {
    const matches = root.querySelectorAll(selector);
    return matches.length > 0 && matches.length <= limit;
  } catch (_) {
    return false;
  }
}
For stable selector:
if (isUniqueSelector(candidateSelector, document, 1)) {
  return {
    selector: candidateSelector,
    unique: true,
    strategy,
    diagnostics
  };
}
Do not accept non-unique selectors as stable.

28. Rect bucket
Do not store absolute pixel coordinates.
Use:
function buildRectBucket(rect) {
  const vw = Math.max(1, window.innerWidth || document.documentElement.clientWidth || 1);
  const vh = Math.max(1, window.innerHeight || document.documentElement.clientHeight || 1);

  const centerX = rect.left + rect.width / 2;
  const centerY = rect.top + rect.height / 2;

  const xPct = centerX / vw;
  const yPct = centerY / vh;

  const xBucket = Math.max(0, Math.min(4, Math.floor(xPct * 5)));
  const yBucket = Math.max(0, Math.min(4, Math.floor(yPct * 5)));

  const horizontal =
    xPct < 0.33 ? 'left' :
    xPct > 0.66 ? 'right' :
    'center';

  const vertical =
    yPct < 0.33 ? 'top' :
    yPct > 0.66 ? 'bottom' :
    'middle';

  return {
    xBucket,
    yBucket,
    region: `${vertical}-${horizontal}`
  };
}

29. Fingerprint similarity
Codex must implement this algorithm.
function calculateFingerprintSimilarity(a, b) {
  if (!a || !b) return 0;

  const fields = [
    { key: 'tagName', weight: 2 },
    { key: 'role', weight: 1 },
    { key: 'type', weight: 1 },
    { key: 'ariaLabel', weight: 2 },
    { key: 'title', weight: 1 },
    { key: 'placeholder', weight: 2 },
    { key: 'dataTestId', weight: 2 },
    { key: 'parentTag', weight: 1 },
    { key: 'parentRole', weight: 1 },
    { key: 'textHash', weight: 1 },
    { key: 'nearbyTextHash', weight: 1 },
    { key: 'siblingButtonCount', weight: 1 }
  ];

  let matched = 0;
  let total = 0;

  for (const { key, weight } of fields) {
    const av = a[key];
    const bv = b[key];

    if (av == null && bv == null) continue;

    total += weight;

    if (String(av || '') === String(bv || '')) {
      matched += weight;
    }
  }

  total += 1;
  if (Array.isArray(a.classTokens) && Array.isArray(b.classTokens)) {
    const setA = new Set(a.classTokens);
    const setB = new Set(b.classTokens);
    const intersection = [...setA].filter(x => setB.has(x)).length;
    const union = new Set([...setA, ...setB]).size || 1;
    if (intersection / union >= 0.5) matched += 1;
  }

  total += 1;
  if (Array.isArray(a.parentClassTokens) && Array.isArray(b.parentClassTokens)) {
    const setA = new Set(a.parentClassTokens);
    const setB = new Set(b.parentClassTokens);
    const intersection = [...setA].filter(x => setB.has(x)).length;
    const union = new Set([...setA, ...setB]).size || 1;
    if (intersection / union >= 0.5) matched += 1;
  }

  total += 1;
  if (rectBucketSimilar(a.rectBucket, b.rectBucket)) {
    matched += 1;
  }

  return total ? matched / total : 0;
}
function rectBucketSimilar(a, b) {
  if (!a || !b) return false;

  return (
    Math.abs((a.xBucket || 0) - (b.xBucket || 0)) <= 1 &&
    Math.abs((a.yBucket || 0) - (b.yBucket || 0)) <= 1 &&
    a.region === b.region
  );
}
Default required similarity:
0.55
If known UI revision mismatch:
0.75

43. Text stability algorithm
Implement:
function isTextStable(previous, current, stableSince, stableMs) {
  const currentHash = shortHash(normalizeText(current || ''));
  const previousHash = previous?.hash || null;
  const previousLength = previous?.length || 0;
  const currentLength = normalizeText(current || '').length;

  if (
    currentHash !== previousHash ||
    Math.abs(currentLength - previousLength) > 2
  ) {
    return {
      stable: false,
      hash: currentHash,
      length: currentLength,
      stableSince: Date.now()
    };
  }

  return {
    stable: Date.now() - stableSince >= stableMs,
    hash: currentHash,
    length: currentLength,
    stableSince
  };
}
Text is stable only if:
- normalized text hash is same
- text length is same or near-same
- no relevant mutation
- stableMs elapsed
Near-same length rule:
Math.abs(current.length - previous.length) <= 2

ResponseLifecycleDetector must not call SelectorResolverV2.resolveComposer() or SelectorResolverV2.resolveSendButton() on every polling tick.

Lifecycle polling interval is typically 600ms.

   - run lightweight composer/send readiness checks

   - optionally run SelectorResolverV2.resolveSendButton()

56. Answer fallback adapter
Run answer fallback only if current extraction fails or answer length < 80.
async function resolveAnswerWithV2Fallback(existingExtractor, existingSelectors = [], meta = {}) {
  let answer = '';
  let answerEl = null;

  try {
    const existing = await existingExtractor?.();
    answer = typeof existing === 'string' ? existing : existing?.text || '';
    answerEl = existing?.element || null;
  } catch (_) {}

  const trimmed = String(answer || '').trim();

  if (trimmed.length >= 80) {
    return { text: trimmed, element: answerEl, source: 'existing' };
  }

  const resolver = window.LLMExtension?.SelectorResolverV2;
  const lifecycle = window.LLMExtension?.ResponseLifecycleDetector;

  if (!resolver?.resolveLatestAssistantAnswer) {
    return { text: trimmed, element: answerEl, source: 'existing_short' };
  }

  const result = await resolver.resolveLatestAssistantAnswer({
    modelName: MODEL_NAME,
    selectors: existingSelectors,
    reason: 'collect',
    traceId: meta?.traceId || meta?.dispatchId || null
  });

  if (result?.ok && result.element) {
    lifecycle?.registerAnswerCandidate?.({
      modelName: MODEL_NAME,
      element: result.element,
      method: result.method,
      selector: result.selector,
      score: result.score,
      confidence: result.confidence,
      traceId: meta?.traceId || meta?.dispatchId || null,
      observedAt: Date.now()
    });

    const text = extractSafeVisibleText(result.element);
    return {
      text,
      element: result.element,
      source: 'selector_resolver_v2',
      partial: text.length < 80
    };
  }

  return { text: trimmed, element: answerEl, source: 'fallback_failed' };
}
